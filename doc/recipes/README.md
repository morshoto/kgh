# Recipes

Practical, copy-adaptable examples for running `kgh` in a real repository.

## Available Recipes

- [Competition Submission Loop](./competition-submission-loop.md): a minimal target for dry-run, live run, and submission-oriented iteration

## How to Use These Recipes

- copy the relevant target shape into `.kgh/config.yaml`
- replace placeholder Kaggle identifiers, notebook paths, and datasets with your own
- run the dry-run command first and inspect the resolved JSON before switching to live mode
